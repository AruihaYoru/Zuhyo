# ズヒョウ（図描）
独自の描画用言語です。

# 前提
長さの単位はなし。角度の単位は度が基本、かなあ。
0度は真東を意味します。1度上がるごとに反時計回りです。
全てはトーンで描画され、線の太さといったプロパティもありません。

## 「構成」定義
<br>**書き方:**<br>
例としてあげるのならば
```text
ID = hoge
// 一行コメントはスラッシュ2個です。
```
まず、先頭にこの構成のIDを宣言します。`ID`は予約済です。
```text
90a1 = b
```
これは、aの90度目（真上）から長さ1の距離を測る、の意味です。そして、その場所をbと定義しています。
<br>また、相対角度／相対距離で記述することも可能です。
```
90a1 = b
+90c+0 = d
```
これは、a~bの時に使用された90度、長さ1の相対です。その時に使用された角度と距離に加算がされます。
この場合、180(90+90)度に長さ1(1+0)、と同義です。

```text
a <-> b
```
これは、aとbの間に実線を引くことを意味します。ほかにも、`<..>`で点線、それこそ`<-->`で破線、`<-.->`で1点鎖線、などでしょうか。もちろん、`<-..->`で2点鎖線ですね！
もちろんカスタムの線を引くこともできます。定義されていないプロパティなど。極論、`<これはカスタムの線です>`と打てば例外処理が走りそう描画されます。

<br>**応用:**<br>
```text
45o0.5 = a
// 原点Ｏの右下315度目から、0.5の距離に点Ａを定義。

90a1 = b
0b1 = c
270c1 = d
```
これは、四角形abcdの各点を指定しています。原点Oは予約済です。
もっとスマートに書くなら:
```text
315o0.5 = a
90a1 = b
+90b1 = c
+90c1 = d
```
とも書けますね。

```text
a <-> b
b <-> c
c <-> d
d <-> a
```
これは、点a~点dを通る道筋に実線を引く、の意味です。
```text
a <-> b <-> c <-> d
```
と省略することもできます。
こうすることで、原点Ｏを中心とした一辺の長さが1の正方形、が定義されます。

### 全文
```text
ID = square

315o0.5 = a
90a1 = b
+90b1 = c
+90c1 = d

a <-> b <-> c <-> d
```
円についてはこう書きます:
```text
ID = circle
0o1 = a
180o1 = b

a <->(50%, 90, 1) b <->(50% 270, 1) a
```
こうすることで、上下の半円が組み合わさって完璧な円になります。`()`内で指定されたことを使用した円弧を描きます。<br>
基本的なルールとしては:
- **第一引数:**線の何%地点で、かを定義します
- **第二／第三引数:**第一引数で指定された位置からどれほどずれた座標か、です。`(arg1, arg2) = (angle, distance) = (0, 1)`です。
<br>ベジェ曲線っぽくもありますかね....ちなみに、三次曲線以上はカッコを複数に増やし指定します（`a <->(33%, 90, 1)(66%, 270, 1) b`など）
また、100%を超えた指定も可能です。（`(150% 90, 10)`で、大きくそれた曲線を作成できます。）ちなみにですが、このパーセントは「実行率」でもあります。a~bの描画中の、75%、といった感じで。

### 円柱について考えてみましょう
`ID = cylinder`として。<br>
まず、原点Oの左右にaとbを定義。
```text
0o1 = a
180o-1 = b 
```
そしてそこから、線を引きます。
```text
a <->(50%, 90, 0.5) b
b <->(50%, 270, 0.5) b 
```
少しだけつぶれていることを表現できます。そして、真下の面を作りましょう。
```text
270a3 = c
270b3 = d
```
aとbの真下から3離れた場所に二つの点を定義します。その後、
```text
c <->(50%, 90, 0.5) d
 ```
で下のふくらみを作成します。
<br>**全文:** 
```text
ID = cylinder
0o1 = a
180o1 = b

a <->(50%, 90, 0.5) b
b <->(50% 270, 0.5) b 

270a3 = c
270b3 = d
c <->(50%, 90, 0.5) d
d <.>(50%, 270, 0.5) c

a <-> c
b <-> d
```
三次元空間的なふくらみを、二次元で強制的につくることができます。

### 高度な記述
**基本的に、上から順番に描画が実行されます。ので、未定義の点や変数を使用するとエラーが発生します。**<br>

`0o0 = center`と記述すれば、centerを原点Oとして扱うことができます。<br>
**引数について:**
<br>もちろん、宣言の形をとっているからには引数はあります。
```text
ID = circle([radius])

0o[radius] = a
180o[radius] = b

a <->(50%, 90, 1) b <->(50% 180, 1) a
```
とすることで、radiusに代入された半径の円が作成できます。<br>
**演算について:**
基本的に、`{}`で実行します。
```text
ID = circle([dia])
dia = {[dia] / 2}

0o[dia] = a
180o[dia] = b
a <->(50%, 0, 1) b <->(50% 180, 1) a
```
これは、直径を基に半径を割り出し円を作成するものです。
sin, cos, tan、その他もろもろを使用することも可能です。その場合は、`{sin(num)`など....<br>
**点位置の指定:**<br>
曲線を描く際、`(50%, 0, 1)`と書きました。これは、`=`にも使用できます。
```text
a~(50%, 0, 0)b = [midpoint]
[midpoint] = c
```
など. ちなみにですが、`a~b`は辺ab、という意味でもあります。変数`midpoint`が示している位置は辺abの中点ですね！

### トーン
各アイテムに対し、`fill`プロパティを指定することでクロージャ・閉領域を算出し完結します。<br>
```text
ID = square

315o0.5 = a
90a1 = b
+90b1 = c
+90c1 = d

a <-> b <-> c <-> d
fill: dot
```
のように。因みにですが、fillは、**「ひとつまえのブロックに記述されている`<->`」で囲まれた場所のみ**を埋め尽くします。<br>
ここで思い出してほしいのが、`<>`の存在です。これは、いうなれば無線ですが、「存在は」しています。
```text
ID = cylinder
0o1 = a
180o1 = b

a <->(50%, 90, 0.5) b
b <->(50% 270, 0.5) a
fill: none
// この言語では、改行と空白が正当な区切りとして使用されます。
// 「空行が一つ挟まれるまで」fillプロパティはそれを一つの区切りとして認識します。
// よって、このfillは「a <->(50%, 90, 0.5) b」と「b <->(50% 270, 0.5) a」で囲まれた場所を埋め尽くします。

270a3 = c
270b3 = d

c <->(50%, 90, 0.5) d
d <>(50%, 270.-0.5) c
fill: line

a <-> c
b <-> d
b <>(50%, 90, 0.5) a
d <>(50%, 270.0.5) c
fill: cross
// b~d、d~cについてもう一度記述していますが、「上書き」などということにはならず重複して描画されます
```
また、トーンについてはオフセットと密度を持っています。
`cross(arg1, arg2, arg3) = (Xoffset, Yoffeset, dencity)`です。
## 制約
- 点の名前に数字は利用できません:`[angle][point][distance]`という都合上、点の名前に数字を使われるとangleあるいはdistanceの一部として解釈されかねません。
- スペース及び改行は正当な区切りです。`a<->b`という記述はできず、`a <-> b`である必要があります。
- 変数名は`[]`で囲まなければなりません:パーサーのためです。また、点の名前と同様に数字を使うことはできません。
- もちろんですが、すべての演算において`{}`を記述しなければなりません:これは描画エンジンであり、数学用ではありませんので。
- 実行順序。もし`a <-> b`を描画する際に b がまだ定義されていなかった場合、描画エンジンはその構造全体を無視します。
